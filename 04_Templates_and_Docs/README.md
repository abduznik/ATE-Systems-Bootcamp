# תיקייה 04: תבניות עבודה ומסמכי תיעוד אדריכלי

תיקייה זו אינה דורשת ביצוע משימה פעילה, אלא משמשת כספריית תשתית לפרויקטים העתידיים שלכם. תיעוד הנדסי הוא החוצץ בין תחביב למקצוע. 

יש להשתמש בתבניות הבאות בכל פיתוח של מערכת חדשה:

1. תבנית רשימת חומרים (BOM_Template.csv):
   כל רכיב חומרה שנדרש לפרויקט חייב להיות מתועד כאן במדויק. כולל מק"ט יצרן, סוג מארז (לדוגמה Through-Hole), וכמות נדרשת. אין לרכוש ציוד ללא BOM מאושר.

2. תבנית אישור קוד (PR_Template.md):
   כאשר תרצו למזג קוד חדש שכתבתם אל תוך הגרסה הראשית של הפרויקט (Main Branch), עליכם לפתוח Pull Request. תבנית זו מכילה רשימת תיוג (Checklist) שמוודאת שבדקתם את כל מערכות הבטיחות ושהקוד שלכם לא פוגע במערכות קיימות. האדריכל לא יבדוק קוד שלא עומד בתבנית זו.

3. שרטוטי ארכיטקטורה (Pinout_Diagram_Blank.drawio):
   תבנית ריקה בפורמט Draw.io לשרטוט חיווטי מחברים ותרשימי זרימה לוגיים. התבנית כוללת חותמת שרטוט תקנית בפינה התחתונה לאחריות מקצועית.

אנא העתיקו את התבניות הללו אל תיקיות הפרויקט שלכם עם תחילת העבודה על פרויקטי חומרת האמת (ATE).
