# משימה 02: סימולציית קושחה ותקשורת טורית

פיתוח ישירות על חומרה ללא סימולציה מוביל לרוב לשריפת רכיבים ולזבוז זמן מעבדה יקר. משימה זו תרגיל אתכם בכתיבת קוד לבקר ESP32 תוך שימוש בסביבת סימולציה וירטואלית.

## מטרת המשימה
שינוי לוגיקת ההפעלה של נורית חיווי כך שתגיב אך ורק לפקודות ספציפיות המגיעות מתקשורת טורית (Serial Communication).

## שלבי ביצוע:
1. פתיחת הסימולטור: היכנסו לאתר Wokwi ופתחו פרויקט ESP32 חדש, או פתחו את קובץ ה-`diagram.json` המצורף בתיקייה זו אם הוא קיים.
2. העלאת הקושחה: העתיקו את קוד ה-C++ מקובץ ה-`Blink_and_Serial.ino` אל עורך הקוד בסימולטור. 
3. חקר הקוד הקיים: שימו לב שהקוד מגדיר חיבור טורית בקצב 115200 באוד (Baud) ומהבהב נורית באופן אוטומטי.
4. שינוי לוגיקה: שנו את הפונקציה `loop()`. עליכם לתכנת את הבקר כך שהנורית תהבהב פעם אחת אך ורק כאשר המשתמש מקליד את האות "A" (גלויה וגדולה) ב-Serial Monitor.
5. אימות תקינות: הריצו את הסימולציה ווודאו שהמערכת יציבה ולא מגיבה לתווים אחרים.
