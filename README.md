# מחנה אימונים למערכות ATE

ברוכים הבאים לארגז החול (Sandbox) של צוות פיתוח חומרה ותוכנה (ATE).
לפני שניגשים לפתח מערכות דיאגנוסטיקה לחומרת אמת במעבדה, עליכם להוכיח שליטה בסביבת הפיתוח, בשיטות העבודה שלנו, ובבקרת תצורה (Version Control). 

מאגר זה נועד להעביר אתכם טירונות טכנולוגית קצרה. כל תיקייה כאן מכילה משימה שנועדה להכין אתכם לפרויקטי הגמר שלכם.

---

## שלב א': התקנת סביבת העבודה (Toolchain)
הצעד הראשון שלכם כמפתחים הוא להכין את עמדת העבודה. התקינו את התוכנות והדרייברים הבאים לפי הסדר:

### תוכנות ליבה:
1. **[GitHub Desktop](https://desktop.github.com/download/)** - לניהול גרסאות וסנכרון הקוד למאגר (כולל התקנת Git אוטומטית).
2. **[Arduino IDE 2.x](https://www.arduino.cc/en/software/)** - סביבת הפיתוח בה נכתוב C++ לבקרי ה-ESP32.
3. **[KiCad 8.0](https://www.kicad.org/download/)** - התוכנה בה נשתמש לתכנון סכמות חשמליות ומעגלים מודפסים (PCB).
4. **[Draw.io Desktop](https://github.com/jgraph/drawio-desktop/releases/tag/v29.7.9)** - ליצירת תרשימי זרימה, בלוקים וארכיטקטורת תוכנה (הורידו את קובץ ההתקנה ל-Windows).

### מנהלי התקן (Drivers) לתקשורת USB חומרה:
כדי שהמחשב שלכם יזהה את בקרי ה-ESP32 שתקבלו בהמשך, חובה להתקין את שני מנהלי ההתקן הבאים:
* **[Silicon Labs CP210x Driver](https://www.silabs.com/software-and-tools/usb-to-uart-bridge-vcp-drivers?tab=downloads)** (נפוץ בלוחות פיתוח מקוריים).
* **[CH340/CH341 USB Driver](https://www.wch-ic.com/downloads/CH341SER_ZIP.html)** (נפוץ בלוחות פיתוח תואמים).

### ספריות Arduino נדרשות (Library Manager):
יש להתקין את הספריות הבאות דרך ה-Library Manager בתוך ה-Arduino IDE:
* **ArduinoJson** (לניהול נתונים מורכבים).
* **SimpleTimer** או שימוש ב-`millis()` (כפי שמתורגל במשימה 05).

## שלב ב': משימות ה-Bootcamp
לאחר שהמחשב שלכם מוכן, היכנסו לתיקיות הבאות לפי הסדר ובצעו את המשימות המפורטות בתוכן:

* 📁 **[משימה 01: יסודות Git ובקרת תצורה](./01_Git_Basics/Welcome_and_Guide.md)**
  * המטרה: לשכפל את המאגר הזה אליכם למחשב (Clone), לעדכן את קובץ רישום הסטודנטים, ולדחוף את השינויים (Commit & Push) כדי להתגבר על הפחד מגיט.

* 📁 **[משימה 02: סימולציית קושחה (Firmware)](./02_Firmware_Simulation/)**
  * המטרה: כתיבת קוד ראשוני ב-C++ ושימוש בסימולטור Wokwi. יצירת לוגיקה שמדליקה נורית חיווי אך ורק כתגובה לפקודה ספציפית מה-Serial Monitor.

* 📁 **[משימה 03: תכנון חומרה ב-KiCad](./03_Hardware_Design/README.md)**
  * המטרה: היכרות עם עולם ה-EDA. שרטוט סכמה ראשונית של בקר ESP32 מחובר למגבר שרת (LM358) והרצת בדיקת חוקיות חשמלית (ERC).

* 📁 **[04: תבניות ומסמכי תיעוד (לשימוש עתידי)](./04_Templates_and_Docs/)**
  * תיקייה זו מכילה את התבניות שבהן תשתמשו בפרויקטים האמיתיים שלכם: רשימת רכיבים (BOM), תבנית לבקשת אישור קוד (Pull Request), ותבניות אדריכלות. הציצו בהם כדי להבין את הסטנדרט המצופה.

* 📁 **[משימה 05: אתגרי לוגיקה וקוד (C++)](./05_Logic_Challenges/README.md)**
  * המטרה: כתיבת קוד מקצועי ללא `delay()`. מימוש מכונות מצבים, עבודה עם ביטים (Bit Masking), וטיפול ברעשים בלחצנים (Debounce) - הכנה קריטית למערכות זמן-אמת.
