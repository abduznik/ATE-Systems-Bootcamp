# משימה 05: אתגרי לוגיקה וקוד (Arduino C++)

הגעתם לשלב הקוד המקצועי. במערכות ATE, אנחנו לא משתמשים בפונקציית `delay()`. 
למה? כי בזמן שהבקר "מחכה", הוא לא יכול לקרוא חיישנים, לבדוק תנאי בטיחות או להגיב למשתמש.

## הקבצים בתיקייה זו:

1. **`State_Machine_Skeleton.ino`**: שלד למימוש מכונת מצבים (State Machine). עליכם לממש מעבר בין מצבי "מנוחה", "בדיקה" ו"שגיאה" על סמך קלטים.
2. **`Bit_Masking_Skeleton.ino`**: תרגול עבודה עם ביטים. במקום לשנות כל פין בנפרד, נלמד כיצד לשלוט על קבוצת פינים בו-זמנית בעזרת מסכות ביטים (AND/OR/XOR).
3. **`Debounce_Skeleton.ino`**: מימוש מנגנון מניעת רעידות (Debounce) ללחצן פיזי ללא שימוש ב-`delay`, אלא באמצעות מדידת זמן (`millis`).

## דגשי פיתוח:
- שימוש ב-`millis()` לניהול זמנים.
- קוד נקי ותיעוד פעולות.
- הפרדה בין לוגיקה למימוש חומרה.
