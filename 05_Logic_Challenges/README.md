# משימה 05: אתגרי לוגיקה וקוד (Arduino C++)

הגעתם לשלב הקוד המקצועי. במערכות ATE, אנחנו לא משתמשים בפונקציית `delay()`. 
למה? כי בזמן שהבקר "מחכה", הוא לא יכול לקרוא חיישנים, לבדוק תנאי בטיחות או להגיב למשתמש.

> [!IMPORTANT]
> **שימו לב:** הקבצים בתיקייה זו הם שלדים (Skeletons) בלבד. הקוד לא יתפקד במלואו ולא "יעשה כלום" עד שתממשו בתוכו את הלוגיקה הנדרשת לפי ההוראות בכל קובץ.

## הקבצים בתיקייה זו:

1. **`State_Machine_Skeleton.ino`**: שלד למימוש מכונת מצבים (State Machine).
   * **המשימה:** ממשו את הלוגיקה ב-`loop`. המערכת צריכה לעבור ל-`TESTING` ל-5 שניות כשלוחצים 's', ואז לחזור ל-`IDLE`. אם נלחץ 'e' בכל שלב - המערכת ננעלת ב-`ERROR`.
2. **`Bit_Masking_Skeleton.ino`**: תרגול עבודה עם ביטים.
   * **המשימה:** השתמשו באופרטורים `^` (XOR), `&` (AND) ו-`|` (OR) כדי לשלוט בביטים בתוך `systemRegister`. אסור להשתמש ב-`digitalWrite`.
3. **`Debounce_Skeleton.ino`**: מימוש מנגנון מניעת רעידות (Debounce).
   * **המשימה:** ממשו את הלוגיקה כך שרק לחיצה אמיתית (יציבה ל-50ms) תדפיס "Button Pressed" ל-Serial.

## דגשי פיתוח:
- שימוש ב-`millis()` לניהול זמנים (בלי `delay`!).
- קוד נקי ותיעוד פעולות.
- הפרדה בין לוגיקה למימוש חומרה.
