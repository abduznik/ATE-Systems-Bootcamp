# ברוכים הבאים לצוות הפיתוח: מדריך כניסה ל-ATE Systems Bootcamp

שלום לכולם וברוכים הבאים!  
הצטרפותכם לצוות הפיתוח היא צעד משמעותי. כדי להבטיח כניסה חלקה ומקצועית לעולם החומרה והמערכות המשולבות (Embedded Systems), הקמנו את מאגר ה-**ATE-Systems-Bootcamp**.

מאגר זה משמש כ"ארגז חול" (Sandbox) – סביבת עבודה בטוחה שבה תוכלו להתנסות, לטעות וללמוד את כלי העבודה שלנו, לפני שנעבור לעבודה ישירה על חומרת מעבדה יקרה ורכיבים רגישים. כאן תלמדו איך אנחנו מנהלים קוד, מתעדים שינויים ומשתפים פעולה בצוות.

---

## 1. למה ניהול גרסאות (Version Control)?

בעבודה על פרויקטים הנדסיים מורכבים, כמעט אף פעם לא עובדים לבד. ניהול גרסאות באמצעות **Git** מאפשר לנו:
* **לעקוב אחרי שינויים:** לדעת בדיוק מי שינה מה ומתי.
* **חזרה לאחור:** אם עדכון מסוים "שבר" את המערכת, ניתן לחזור לגרסה יציבה בלחיצת כפתור.
* **עבודה במקביל:** מספר מהנדסים יכולים לעבוד על חלקים שונים של אותו קוד מבלי לדרוס אחד את השני.

### התקנת סביבת העבודה (Windows)
כדי לפשט את העבודה, אנו משתמשים ב-**GitHub Desktop**. זהו ממשק גרפי נוח שהופך את העבודה עם Git לוויזואלית ופשוטה.
1. הורידו את המתקין מהאתר הרשמי: [desktop.github.com](https://desktop.github.com/).
2. הריצו את קובץ ה-`.exe` ועקבו אחר ההוראות.
3. **חשוב לדעת:** התקנת GitHub Desktop כוללת בתוכה את מנוע ה-Git "מאחורי הקלעים", כך שאין צורך להתקין את Git בנפרד.

## 2. צעדים ראשונים: המשימה שלכם

עקבו אחר השלבים הבאים כדי לבצע את הפעולה הראשונה שלכם במאגר:

### שלב א': ביצוע Clone (שכפול המאגר)
כדי להתחיל לעבוד, עליכם להוריד עותק מקומי של המאגר למחשב שלכם.
1. פתחו את GitHub Desktop.
2. התחברו לחשבון ה-GitHub שלכם (במידה ואין לכם, צרו אחד).
3. בחרו באופציה **Clone a Repository from the Internet**.
4. חפשו את `ATE-Systems-Bootcamp` תחת הארגון שלכם ובחרו לאן לשמור אותו במחשב (נתיב ללא רווחים בעברית מומלץ).

### שלב ב': עריכת קובץ התיעוד
1. נווטו בתיקיית המחשב אל: `01_Git_Basics/students_log.md`.
2. פתחו את הקובץ באמצעות עורך טקסט (Notepad++, VS Code, או אפילו Notepad רגיל).
3. הוסיפו שורה חדשה לטבלה עם השם שלכם, התאריך ופרט טריוויה מעניין.
4. שמרו את הקובץ (`Ctrl+S`).

### שלב ג': ביצוע Commit (הצהרה על שינוי)
חזרו ל-GitHub Desktop. אתם תראו שהתוכנה זיהתה ששיניתם את הקובץ.
1. מצד שמאל למטה, תראו שדה בשם **Summary**.
2. כתבו הודעה קצרה והגיונית, למשל: `feat: add [Your Name] to students log`.
3. לחצו על הכפתור הכחול: **Commit to main** (אל דאגה, בשלב הבא נעביר את זה לענף משלכם).

### שלב ד': יצירת Branch וביצוע Push
אנחנו לעולם לא דוחפים שינויים ישירות לגרסה הראשית מבלי לבדוק אותם.
1. לחצו על **Current Branch** בחלק העליון של המסך.
2. בחרו **New Branch**. קראו לו בשם: `feat/add-name-[your-name]`.
3. ודאו שאתם על ה-Branch החדש שיצרתם.
4. לחצו על **Publish branch** (או **Push**). פעולה זו מעלה את השינויים שלכם מהמחשב המקומי אל השרת בענן.
